About the app

This app - lets you take an IBM Domino discussion database with you on your Android device.
The content is stored locally on the device, letting you read and participate in the discussion even if there is no network connection. When you do have a network connection, the database replicates new entries from the server to the app, and any entries written by you on the device get replicated to the Domino server. Replication happens in the background on a schedule configured by you.
You can configure the app to replicate with multiple discussion databases.

Getting the app to work

Server and administration

The Domino Data Service api is used to replicate with IBM Domino. This requires the IBM Domino server to be either release 9+ or it can be release 8.5.3+ with the Extension Library or Upgrade Pack. It also requires the server and discussion database to be configured to allow the use of the Domino Data Service api.
You need to talk to your Domino administrator to make the Domino Data Service available. Further information can be found in Server configuration.

App configuration

When the app launches, you have a Configuration menu item. Click it to configure.
When you get to the Database configurations view, you can click + action button to add a database that you would like to make available in the app.
You can also click the Settings action button to configure replication intervals and if you want to enable extra logging. Replicating often will drain your battery.

Configuring a new discussion database

Be aware that there is very little quality checking in the form that you fill out when configuring replication with database. A very common issue is that your device keyboard might insert a space character in places where they were not meant to be. For instance many keyboards insert a space character when you put a punctuation mark in a text field.

Discussion database name

This is what you would like to call the database when displayed in the app.

Host name

The Internet DNS name of the server on which the database is located. Could be something like mail.acme.com or www.databases.acme.com.

Path to database

This is the path to the database when it is located on the server. Values could be for instance /discussions/salesdisc.nsf or /sales/news.nsf.

Username for login

This is your Notes name. The name you see when creating emails in Notes for instance. Common values look like this: Jens Bruntt/Sales/Acme or Jack Johnson/Acme.

Password

This is your http password. This is not always the same value as when you use your Notes client. If you use iNotes, Traveler or web applications located on IBM Domino, then the password you should configure is the one you use in those applications.

Use SSL

Check to enable SSL. Encrypts data sent to and retrievede from the server. Highly advisable to use.

HTTP Port

This is the port used to communicate with the server. It is a number. Usually 80 (for non-ssl) or 443 (for ssl). If you check the use ssl option, you most likely should put 443 here.

Log

There is a log available in the app from the action bar. How much is written to the log depends on your configuration.
Note that you can delete the log contents, using the delete action bar button.

Limitations

  • Only Domino Discussion Databases made with the standard template (StdR9Discussion) have been tested with the app. Other disucssion database designs may work.
  • Images are not displayed.

Security

When you configure the app, please go for SSL (also known as HTTPS). Otherwise your username and password will get transmitted over the Internet in a form that is easy to get access to.
Be aware that your username and password do get stored in the app. If you lose the device, the password is not easily accessible, but someone with the right tools will be able to find and copy it.
Make sure that you use a Screen lock and a good device retrieval app that can help you wipe apps and data if you lose it.
You should consult with the server administrator in your organisation before using this app, to make sure that it meets IT-guidelines.
